# Security Policy

## Supported Versions

Only the latest release of DMS is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| Main / Latest |  Yes |
| < Latest |  No |

## Reporting a Vulnerability

We take the security of DMS seriously. If you find a security vulnerability, please do not disclose it publicly. Instead, report it using the following procedure:

1.  Send an email to the project maintainers (or open a private security draft on GitHub if available).
2.  Provide a detailed description of the vulnerability, including:
    *   The affected component/route.
    *   Steps to reproduce the vulnerability (including payloads or proof-of-concepts).
    *   Potential impact.
3.  We will acknowledge your report within 48 hours and work with you to resolve the issue.

Please do not open public issues for security vulnerabilities.
